## Project Overview

Hinario API is an ASP.NET Core 10 Web API for managing a hymnal (hinário), with PostgreSQL as the database. It includes full-text search, hymn navigation, web scraping from an external hymn source, and bulk import.

## Common Commands

```bash
# Run locally
dotnet run

# Build
dotnet build

# Run with Docker (API + PostgreSQL)
docker compose up --build

# Apply EF Core migrations manually
dotnet ef database update

# Add a new migration
dotnet ef migrations add <NomeDaMigration>

# Restore packages
dotnet restore
```

## Architecture

The project follows a layered architecture:

- **Controllers/** — HTTP endpoints, minimal logic. `HinoController` maps to `api/hino`.
- **Repositories/** — Data access via EF Core. Implement interfaces defined in `Interfaces/`.
- **Services/** — External integrations (e.g., web scraping). Injected via `IHttpClientFactory`.
- **Context/** — `HinarioApiContext` (EF Core DbContext), single `DbSet<Hino>`.
- **Models/** — `Hino` entity + several DTOs (`HinoDto`, `HinoResultadoPesquisaDto`, `HinoImportDto`).
- **Utils/** — `TextNormalizer` (accent removal) and `TrechoPesquisaHelper` (search preview builder).

Dependency injection is wired in [Program.cs](Program.cs): `IHinoRepository` → `HinoRepository`, `ICampinaGrandeMineracaoService` → `CampinaGrandeMineracaoService`.

Auto-migration runs on startup (via `db.Database.Migrate()`) — this is intentional for Docker deployments.

## Key Domain Details

**Hymn identifiers** follow the pattern `TYPE-NUMBER` (e.g., `H-1`, `C-5`). Navigation endpoints (`/proximo`, `/anterior`) use this structure. The defined type ordering is: `H`, `C`, `S`, `HC`, `L`.

**Full-text search** uses PostgreSQL `tsvector` with a GIN index on the `letra_idx` column (Portuguese language config). The column is generated by the database — never set it manually. Search results include a `Trecho` preview with matched terms highlighted in HTML `<b>` tags.

**Web scraping** is handled by `CampinaGrandeMineracaoService`, which fetches hymns from an external church website using `HtmlAgilityPack`. It handles multiple encodings (UTF-8, ISO-8859-1, UTF-16) and normalizes spacing.

## Database

- PostgreSQL 16 (Docker container `hinario-db`, mapped to port `5433`)
- Connection string set via `ConnectionStrings__DefaultConnection` env var in Docker, or in `appsettings.Development.json` locally
- EF Core manages the schema via migrations in [Migrations/](Migrations/)
